# ai-app-saas

## Project Overview
A modern SaaS starter monorepo featuring a React (Vite) frontend, Express backend, and shared TypeScript code. Designed for rapid development, scalability, and best practices in developer experience.

## Monorepo Structure
- `frontend/` – React app (Vite, TypeScript)
- `backend/` – Express API (TypeScript)
- `shared/` – Shared utilities, types, and logic (TypeScript)
- `infra/` – Infrastructure, CI/CD, linting, formatting, and base configs

## Tech Stack
- **Frontend:** React, Vite, TypeScript
- **Backend:** Node.js, Express, TypeScript
- **Shared:** TypeScript utilities and types
- **Tooling:** ESLint, Prettier, Vitest, Docker, GitHub Actions

## Getting Started
1. **Clone the repo:**
   ```sh
   git clone https://github.com/AnouarY02/ai-app-saas.git
   cd ai-app-saas
   ```
2. **Install dependencies:**
   ```sh
   npm install
   ```
3. **Set up environment variables:**
   - Copy `.env.example` to `.env` and fill in required secrets.

## Environment Variables
Required in `.env`:
- `NODE_ENV` (e.g. development, production)
- `JWT_SECRET` (JWT signing secret)
- `SESSION_SECRET` (session secret)
- `PORT` (backend port, default: 3000)

Optional:
- `FRONTEND_URL` (default: http://localhost:5173)
- `BACKEND_URL` (default: http://localhost:3000)
- `LOG_LEVEL` (e.g. info, debug)

## Running Locally
- **Development (hot reload):**
  ```sh
  npm run dev
  ```
  This runs both frontend and backend concurrently.

- **Frontend only:**
  ```sh
  cd frontend && npm run dev
  ```
- **Backend only:**
  ```sh
  cd backend && npm run dev
  ```

## Scripts Reference
- `npm run dev` – Start frontend & backend in dev mode
- `npm run build` – Build all workspaces
- `npm run lint` – Lint all code
- `npm run format` – Format codebase
- `npm run test` – Run tests (Vitest)

## Docker Usage
- **Build and run with Docker Compose:**
  ```sh
  docker-compose up --build
  ```
- **Frontend:** http://localhost:5173
- **Backend:** http://localhost:3000

## API Endpoints
- All backend API routes are served from `/api` (see backend/src/routes/).

## Authentication Flow
- JWT-based authentication. Set `JWT_SECRET` and `SESSION_SECRET` in `.env`.
- Auth endpoints: `/api/auth/*` (see backend routes).

## Development Guidelines
- Use TypeScript everywhere.
- Shared code lives in `shared/` and is imported by both frontend and backend.
- Lint and format before pushing.

## Testing
- Run all tests with:
  ```sh
  npm run test
  ```
- Uses [Vitest](https://vitest.dev/) for unit/integration tests.

## CI/CD
- GitHub Actions workflow: see `.github/workflows/ci.yml`.
- Runs lint, build, and test on every push/PR to `main`.
