### Script Structure (15–30 seconds)

```
[0–3s]  HOOK — pattern interrupt, make them stop scrolling
[3–10s] PROBLEM — name their pain with specificity
[10–20s] REFRAME — give the behavioral science insight (1 sentence)
[20–25s] PROOF — personal result or data point
[25–30s] CTA — one of the 3 variants above
```

**Example script (Hook #7):**
```
[HOOK] Your 3 PM crash is not about food.

[REFRAME] Your biological alertness dips 7–9 hours after waking.

        Crash gone in 4 days.
[CTA] VOLT Sleep gives you 1 action per day to fix this. Free. Link in bio.
```

## PHASE 2 — LANDING PAGE CONVERSION TUNING

### Current Landing Structure Audit

**Current flow:** Badge → Hero (headline + sub + CTA) → Stats → How It Works → Value Props → Pricing → Disclaimer → Footer

**Current headline:** "Energy for ambitious people."
**Current sub:** "Your personal energy coach. Based on behavioral science, built for high performers."
**Current CTA:** "Start free — 30 seconds"

**Strengths:**
- Clear value prop (energy, not sleep)
- Low friction CTA (free, 30 seconds)

## PHASE 3 — 30-DAY TRACTION PLAN

### Funnel Math (Conservative Assumptions)

```
Impressions → Click-through → Landing → Signup → Activated → Converted

Assumptions:
  Short-form CTR:    1.5% (industry avg for niche content)
  Landing → Signup:  8% (good for free product)
  Signup → Activate: 40% (complete onboarding + 1 check-in within 48h)
  Activate → Pay:    5% (within 30 days, conservative)

To reach 100 paying users:
  100 paying = 2,000 activated users
  2,000 activated = 5,000 signups
  5,000 signups = 62,500 landing visitors
  62,500 visitors = 4,166,667 impressions

At 6 posts/day, need ~70K avg impressions per post.
Realistic range: 5K–500K per post (power law distribution).
Expect 20% of posts to drive 80% of traffic.
```

## PHASE 4 — METRIC INTERPRETATION (Diagnostic Playbooks)

### IF Activation < 30%

**Diagnosis:** Users sign up but don't complete onboarding + first check-in within 48h.

**Root cause analysis (check in order):**

| Check | Query | Fix |
|-------|-------|-----|
| Onboarding drop-off step | `onboarding_step_completed` events — which step loses most? | Simplify that step. If step 3+ → move to fewer steps. |
| Instant Value Mode engagement | `instant_value_preview` event count vs signup count | If low → preview card isn't compelling. Change the demo card content. |
| Time to first card | Time between `onboarding_completed` and `daily_card_viewed` | If >1 hour → add immediate redirect to dashboard after onboarding. |
| Email deliverability | Magic link open rate (Supabase auth logs) | If <50% → emails going to spam. Switch to OTP code. |
| Mobile UX | Device breakdown in analytics | If mobile >70% and activation is low → test mobile-specific onboarding. |

**Emergency fix:** Reduce onboarding to 3 steps (merge steps 2+3). Show Daily Card immediately after step 2.

### IF Conversion < 3%

**Diagnosis:** Users activate and use the app but don't pay.

**Root cause analysis:**

| Check | Query | Fix |
|-------|-------|-----|
| Paywall view rate | `paywall_viewed` / `user_activated` | If <20% → users never see the paywall. Add more conversion triggers. |
| Conversion trigger distribution | `conversion_trigger` events by reason | If `card_count` dominates → users don't experience energy improvement. Rules engine may need tuning. |
| A/B test pricing | Experiment results query from OPERATIONS.md | If $14.99 converting at <50% of $9.99 rate → price too high. Revert. |
| Trial engagement | Trial users' `daily_card_viewed` count | If <3 cards viewed during trial → trial too short or not valuable enough. |
| Paywall friction | Time on paywall page → bounce rate | If <10s avg → paywall copy not compelling. Test new sub-headline. |

**Emergency fix:** Add a "blurred insight" on dashboard (already implemented) + show paywall after every 2nd completed action for free users.

### IF D7 Retention < 25%

**Diagnosis:** Users try the product but don't come back after a week.

**Root cause analysis:**

| Check | Query | Fix |
|-------|-------|-----|
| Check-in completion | Morning + night check-in rates for D2–D7 | If declining sharply → check-in is too long or not rewarding. |
| Card quality | Completed actions per user per day | If <0.5 → actions aren't resonating. Review rules engine output variety. |
| Streak engagement | `streak_milestone` events | If no users hitting 3+ day streaks → streak mechanic not visible enough. |
| Micro win detection | `energy_improvement` events | If 0 → users aren't logging enough data for detection. Push morning check-in harder. |
| Re-engagement | `reengagement_sent_at` coverage | If not sending → implement email/push re-engagement. |

**Emergency fix:** Add browser push notification for morning check-in. Simplify evening check-in to 2 fields.

### IF LLM Cost > 15% of Revenue

**Diagnosis:** AI costs are eating margin.

**Action ladder (in order):**

| Step | Action | Impact |
|------|--------|--------|
| 1 | Set `LLM_MODE=reduced` | Daily cards → rules-only. LLM only for weekly + cognitive switch. Cuts ~80% of LLM calls. |
| 2 | Check per-user token usage | `getUserMonthlyTokens()` — find outlier users consuming disproportionate tokens. |
| 3 | Lower monthly token cap | Reduce `MONTHLY_TOKEN_CAP_PER_USER` from 50K to 25K. |
| 4 | Cache weekly insights | Generate weekly report once, serve cached version for 7 days. |
| 5 | Set `LLM_MODE=rules-only` | Kill switch. Zero LLM cost. Product still works (rules engine handles everything). |

**Target economics at 100 paying users:**
```
Revenue:  100 × €9.99 = €999/month
LLM cost: 100 × ~€0.20 = €20/month (2% of revenue)
Margin:   €979/month (98% gross margin)

Even at $14.99 variant:
Revenue:  50 × €14.99 = €750/month
LLM cost: 50 × ~€0.20 = €10/month (1.3% of revenue)
```

## PHASE 5 — BREAKPOINT STRATEGY

### At 100 Paying Users

**Revenue:** ~€1,000/month MRR
**Focus:** Validate retention and unit economics.

| Action | Why |
|--------|-----|
| Lock in pricing experiment winner | Enough data to pick $9.99 vs $14.99. Kill the loser. |
| Measure 30-day retention | Do paying users stay? If <70% M1 retention → product problem. |
| Calculate true LTV | (ARPU × avg months retained). Need: >€30 LTV for sustainability. |
| Document top 3 acquisition channels | Which hooks/platforms drove paying users? Double down. |
| Run NPS or 1-question survey | "What would you miss most if VOLT Sleep disappeared?" → validates core value. |
| Set up Stripe billing alerts | Churn notifications, failed payment recovery. |

**Do NOT:**
- Add features
- Expand to new platforms
- Hire
- Raise prices (yet)

### At 300 Paying Users

**Revenue:** ~€3,000/month MRR
**Focus:** Systematize acquisition and optimize conversion.

| Action | Why |
|--------|-----|
| Productize referral loop | If referral conversion >5% → add better incentives. If <2% → deprioritize. |
| Test Premium Pro tier publicly | Unhide `?pro=1`. Test if 5–10% of users upgrade to $19.99. |
| Add annual plan push | At renewal moments, offer 42% annual discount. Increases LTV. |
| Build email drip sequence | Welcome → D1 tip → D3 check-in reminder → D7 progress → D14 upgrade. Automate. |
| Create 3 "evergreen" content pieces | Blog posts targeting "afternoon energy crash", "caffeine timing", "social jetlag". SEO play for long-term organic. |
| Analyze cohort retention curves | Is D30 retention improving month-over-month? |

**Do NOT:**
- Build a mobile app (web-first until 1,000)
- Add wearable integrations
- Internationalize marketing (keep to 1–2 languages)
- Add team/enterprise features

### At 1,000 Paying Users

**Revenue:** ~€10,000/month MRR
**Focus:** Systemize everything. Prepare for scale.

| Action | Why |
|--------|-----|
| Automate content pipeline | Use `generateEnergyInsight()` API to auto-generate content hooks from user data patterns. Feed back winning angles. |
| Implement proper analytics | Migrate from in-memory telemetry to persistent Supabase analytics. Build real dashboard. |
| Formalize A/B test process | Monthly experiment cadence. 1 pricing test + 1 conversion test + 1 retention test. |
| Evaluate mobile app ROI | Push notifications alone could increase D7 retention 15–20%. Worth the investment at this scale. |
| Consider paid acquisition | With known LTV (€30+), test paid ads at LTV/3 CAC target (€10 CAC). |
| Hire first support/community person | At 1,000 users, support load justifies part-time help. |
| Monthly growth report | Use OPERATIONS.md Section 13 format. Track month-over-month. |

**System checklist at 1,000 users:**
- [ ] Automated weekly growth report (SQL → email)
- [ ] Churn prediction (users with declining check-in frequency)
- [ ] Automated re-engagement emails (3-day, 7-day, 14-day inactive)
- [ ] Revenue alerting (Stripe webhook → Slack for new subscriptions + churns)
- [ ] LLM cost monitoring (daily automated check against thresholds)
- [ ] Content performance tracking (UTM → signup → conversion attribution)
